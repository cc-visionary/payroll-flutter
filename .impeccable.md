## Design Context

### Users

Internal Luxium HR + finance team running payroll for ~30 employees across GameCove, HAVIT, Paper Living, OgKilz, and Luxium HQ. Daily desktop use (macOS / Linux), often back-to-back with Lark, BigSeller, ERPNext.

Primary jobs-to-be-done: review attendance exceptions, approve OT/leave, generate payslips, sync from Lark.

### Brand Personality

Operational. Direct. Transparent. Voice mirrors luxium-website: *"Built from real operational experience"* — confident, no hype.

Three-word personality: **Reliable. Structured. Calm.**

Emotional goals: confidence in numbers, calm in repetitive workflows, zero ambiguity in approval state.

### Aesthetic Direction

Stripe HDS / Linear / Vercel admin dashboards — tight tracking, generous whitespace, soft layered shadows, electric purple as the single CTA accent on a near-white (light) or near-black (dark) field.

Light mode mirrors the website verbatim. Dark mode is a brand-tinted companion (deep navy surfaces, same purple CTA, same green status accent) for all-day data work.

### Design Principles

1. **Purple is the single CTA** — `#635BFF` is reserved for primary actions, links, and focus rings. Don't dilute it on chrome.
2. **Tight headings, calm body** — heading letter-spacing always negative (~-0.04em), body 1.5+ line-height, 16px base.
3. **6px radius baseline** — `lg=6px`, pills (`rounded-full`) only for page-level CTAs and badges. No 12px / 16px pillbox cards.
4. **Status by hue, not by border** — success/warning/error chips use tinted backgrounds with darker text; never colored borders on cards.
5. **Tabular data is monospaced** — Geist Mono for IDs, dates, currencies, and any number that must align across rows.

## Typography

- **Display + Body:** Satoshi (variable 300–700) — bundled from `assets/fonts/Satoshi-Variable.ttf`. Source: Fontshare (TTF mirror of the woff2 already in `luxium-website/public/fonts`).
- **Mono:** Brand spec is **Geist Mono**, but the `google_fonts` package doesn't ship Geist yet. Current substitute: **JetBrains Mono** (closest visual + metric match — geometric, designed for code legibility). Switch to Geist when google_fonts adds it, or bundle `Geist-Mono.ttf` locally and register in `pubspec.yaml`. Used via `AppTheme.mono(context)`.

## Layout & Spacing

- **Base unit:** 4px. Allowed steps: `4 · 8 · 12 · 16 · 24 · 32 · 48 · 64`.
- **Content max-width:** 1100px (matches `lib/widgets/responsive_table.dart`); marketing-style hero sections may go to 1266px.
- **Border radius scale:** `sm=4 · md=5 · lg=6 (default) · xl=8 · 2xl=11 · pill=999`.
- **Card padding:** 16 (compact) / 24 (default) / 32 (hero).

## Elevation

- `shadow-card` (default cards): two-stop soft shadow ≈ `rgba(50,50,93,0.10)` blur 36 / `rgba(0,0,0,0.08)` blur 18.
- `shadow-button` (primary CTA): `rgba(50,50,93,0.20)` blur 24 + `rgba(0,0,0,0.12)` blur 16.
- `shadow-hairline`: `rgba(0,0,0,0.06)` blur 6 — for chips and inline elements.
- **No borders on cards by default** — shadow does the lifting. Use border only on inputs.

## Motion

- **Easing:** `cubic-bezier(0.16, 1, 0.3, 1)` (spring) for enter, `cubic-bezier(0.4, 0, 0.2, 1)` (smooth) for exit.
- **Durations:** 120ms (micro), 200ms (default), 320ms (panel), 480ms (hero).
- Always respect `MediaQuery.disableAnimations`.

## Anti-patterns

- No dark-mode-of-the-website on the website itself; OK to have it in this app.
- No gradient buttons in app chrome. (Gradients reserved for hero cards / charts only.)
- No emoji icons. Use Lucide / Material icons only.
- No multi-color borders on cards. Use shadow + tinted background instead.
- No system fonts in production UI — Satoshi must be loaded before first paint.
